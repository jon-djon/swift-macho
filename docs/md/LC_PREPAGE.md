# LC_PREPAGE

**Command ID:** `0x0A`

**Obsolete.** A marker command that requested pre-paging of mapped file regions. This was intended as a performance hint to the kernel to pre-fault pages into memory. It was never widely used and is not generated by any current toolchain.

The implementation parses only the load command header; no additional fields are defined.

## Fields

| Name | Description | Offset | Size | Type |
|------|-------------|--------|------|------|
| Command ID | Load command identifier (`LC_PREPAGE`) | 0 | 4 | `UInt32` |
| Command Size | Total size of this load command | 4 | 4 | `UInt32` |

**Fixed size:** 8 bytes
