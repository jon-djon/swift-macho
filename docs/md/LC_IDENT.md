# LC_IDENT

**Command ID:** `0x08`

**Obsolete.** Contains null-terminated identity strings describing the binary's origin. This command was used by early Mach-O tools to record identification information (such as the compiler or assembler that produced the object file), but it has not been generated by Apple's toolchain for many years.

The strings are packed consecutively after the load command header, each null-terminated, filling the remaining `cmdsize` bytes.

## Fields

| Name | Description | Offset | Size | Type |
|------|-------------|--------|------|------|
| Command ID | Load command identifier (`LC_IDENT`) | 0 | 4 | `UInt32` |
| Command Size | Total size of this load command (including identity strings) | 4 | 4 | `UInt32` |
| Identity Strings | Consecutive null-terminated identification strings | 8 | variable | `String[]` |

**Minimum size:** 8 bytes (header only, before strings)
