# LC_PREBOUND_DYLIB

**Command ID:** `0x10`

**Obsolete.** Records prebinding information for a dynamic library. When a binary was prebound, the static linker resolved symbol addresses at link time and stored them directly in the binary. At load time, if the library loaded at the expected address, `dyld` could skip binding entirely, improving launch performance.

Prebinding was replaced by the dyld shared cache and chained fixups, which achieve similar or better performance without the fragility of fixed load addresses. This command is not generated by modern toolchains.

The full format includes a library name, module count, and a bit vector indicating which modules are linked, but the implementation currently parses only the load command header.

## Fields

| Name | Description | Offset | Size | Type |
|------|-------------|--------|------|------|
| Command ID | Load command identifier (`LC_PREBOUND_DYLIB`) | 0 | 4 | `UInt32` |
| Command Size | Total size of this load command | 4 | 4 | `UInt32` |
| Name Offset | Byte offset to the library pathname string | 8 | 4 | `UInt32` |
| Num Modules | Number of modules in the library | 12 | 4 | `UInt32` |
| Linked Modules Offset | Byte offset to the linked modules bit vector | 16 | 4 | `UInt32` |

**Minimum size:** 20 bytes (header + fixed fields, before variable data)
